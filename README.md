# cubalizator_bot

Telegram-фактчекер на GigaChat. Отвечает при упоминании `@cubalizator_bot` в любом чате, куда добавлен; при упоминании в ответе на чужое сообщение передаёт цитату в GigaChat.

## Локальная проверка

```bash
python -m venv .venv
# Windows: .venv\Scripts\activate
# macOS/Linux: source .venv/bin/activate
pip install -r requirements-dev.txt
pytest -q
```

Для локального webhook нужен публичный HTTPS URL. Основной вариант развёртывания — Yandex Cloud Serverless Containers.

## Переменные окружения

Скопируйте `.env.example` в `.env` только для локальной работы. `.env` не коммитить.

`ALLOW_REPLY_TO_BOT=false` означает: бот отвечает только при `@упоминании`. Если поставить `true`, он также продолжит разговор при обычном ответе на сообщение самого бота.

## Сборка Docker

```bash
docker build -t cubalizator:latest .
```

## Webhook

После публикации контейнера задайте локально переменные `TELEGRAM_BOT_TOKEN`, `PUBLIC_URL`, `WEBHOOK_PATH_SECRET`, `TELEGRAM_WEBHOOK_SECRET`, затем выполните:

```bash
python scripts/set_webhook.py
```

URL будет таким:

```text
https://<public-container-url>/telegram/webhook/<WEBHOOK_PATH_SECRET>
```

## Поиск по истории группы

Бот умеет отвечать на вопросы про переписку («на какую сумму Бен получил премию»,
«с кем Илюха тусил в Питере», «что Даня говорил про BMW»).

Как это работает:
1. При старте экземпляра функции один раз лениво читается `data/chat_history.json`
   и строится инвертированный индекс в памяти (BM25). Весь JSON в GigaChat не шлётся.
2. Из вопроса определяется человек (с учётом алиасов из `app/aliases.py`) и период
   («в этом году», «за лето», «в январе»). По ним отсекаются кандидаты.
3. По оставшимся сообщениям считается BM25; для денежных вопросов сообщения с
   суммами получают приоритет.
4. Топ-фрагменты (автор, дата, текст) передаются GigaChat как единственный источник.
   Если ничего не найдено — бот отвечает «В переписке этого не нашёл», не выдумывая.

Путь к файлу задаётся переменной `CHAT_HISTORY_PATH` (по умолчанию
`data/chat_history.json`). Если файл отсутствует или повреждён — приложение не
падает, обычные вопросы продолжают работать, а на вопрос по истории бот честно
скажет, что переписку сейчас не поднять.

### Обновить историю из свежего экспорта Telegram

Выгрузи чат через Telegram Desktop в JSON (`result.json`) и собери слим-версию:

```bash
python scripts/build_history.py result.json data/chat_history.json
```

Скрипт выкидывает фото, реакции и метаданные, оставляя автора, дату и текст —
файл становится примерно вдвое легче и быстрее грузится.
